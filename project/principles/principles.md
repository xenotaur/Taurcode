# LRH Principles

## Purpose
These principles define how the repository should be interpreted through the LRH control plane.

## Principles
1. **Prompt-first utility**: prioritize clear, reusable prompt assets and text-replacement workflows that support AI-assisted coding.
2. **Grounded changes**: make repository changes based on observable files and documentation; flag assumptions explicitly.
3. **Non-destructive operations**: prefer additive, auditable updates over broad refactors.
4. **Tool portability**: maintain compatibility with multiple prompt injection/replacement systems, with Espanso currently evidenced.
5. **Traceable execution**: tie current status and planning to concrete evidence artifacts.

## Interpretation Rule
When artifacts conflict, apply LRH precedence: principles → goal → roadmap → focus → work items → guardrails/runtime context.
