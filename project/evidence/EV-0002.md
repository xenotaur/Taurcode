# Evidence Record EV-0002

## Scope

This record summarizes implementation evidence for the adopted Espanso metadata round-trip design proposal.

## Observed Implementation Signals

- Espanso package metadata preservation was implemented for import/export, including reserved package metadata handling in prompt loading.
- Required Espanso defaults were centralized so Markdown-only packages can still export complete Espanso package files.
- Conservative metadata linting was added with separate error and warning results for source and generated packages.
- Merge import semantics preserve curated Markdown prompt frontmatter while updating Espanso-derived keyword and body content.
- Taurcode-written Markdown prompts and generated README output normalize final newlines to reduce non-semantic diffs.
- Prompt-source lint policy and CLI checks were added for canonical prompt packages.
- A conservative prompt formatter was added for explicit, diff-friendly source normalization.
- Merge import preserves existing frontmatter formatting when keyword semantics are unchanged.
